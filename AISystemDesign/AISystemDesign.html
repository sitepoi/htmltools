🔄 Agent Workflow Patterns
👤 Doer Agent
Performs the actual work: drafts documents, runs queries, extracts data. Receives task from orchestrator, executes, and submits for review.
🔍 Reviewer Agent
Reviews Doer output against rules, policies, and quality standards. Flags issues, requests revisions, or approves for manager sign-off.
👔 Manager Agent
Orchestrates the workflow: assigns tasks to Doers, collects Reviewer feedback, makes final decisions, handles escalations to human operators.

⚡ Quick Presets:
📝 Prompts & Agent Persona 💡 This defines WHO the AI is and HOW it behaves. The System Prompt is the AI's "job description" — it sets the rules, tone, and boundaries for every response.
📚 RAG — Knowledge Base 💡 RAG = Retrieval-Augmented Generation. It gives the AI access to YOUR documents so it can answer based on real data — not guessing. Without RAG, the AI only knows what it was trained on. With RAG, it can cite specific policies, standards, and client documents.
🧠 Memory Architecture 💡 Memory is what the AI REMEMBERS across conversations. External Shared Memory (like Redis) means ALL your apps (web chat, VS Code, GitHub Copilot, MS Copilot, custom apps) share the same memory. You ask something in one app — another app remembers it. Think of it as a universal AI brain.
🔌 API, MCP & App Connectors 💡 MCP (Model Context Protocol) = the AI's "hands" — tools it can use to DO things (search, send email, query databases). App Connectors = links to YOUR specific business apps (Ephorm, CaseWare, etc). More tools = more capability BUT also more cost (~$5-8/mo per tool) and larger security surface. Only enable what you need.
📊 Usage & Pilot Scope
🎯 Target Use Cases 💡 What will your AI actually DO? More use cases = more connectors needed = higher cost and more complex architecture. Start with 1-2 use cases for PoC, expand later. Each use case drives which apps to connect, what tools the AI needs, and how much memory/storage is required.
